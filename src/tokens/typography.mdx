# Typography

YongUI의 타이포그래피 시스템은 명확한 정보 계층과 뛰어난 가독성을 제공합니다.
[Pretendard Variable](https://github.com/orioncactus/pretendard)을 주 서체로 사용하며, 체계적인 텍스트 스타일을 제공합니다.

## 타이포그래피 원칙

### 📖 가독성 우선

- **충분한 크기**: 본문은 최소 14px(0.875rem) 이상 사용
- **적절한 행간**: 읽기 편한 1.5배 기본 행간
- **명확한 계층**: 크기, 굵기, 색상으로 정보 계층 구분

### 🎯 일관성

- **통일된 스케일**: 12단계의 체계적인 크기 시스템
- **제한된 스타일**: 필요한 만큼만 스타일 정의
- **예측 가능한 사용**: 동일한 용도는 항상 같은 스타일

### ♿ 접근성

- **최소 크기 준수**: 본문 14px, 캡션 12px 이상
- **충분한 대비**: WCAG 2.1 기준 충족
- **유연한 단위**: rem 단위 사용으로 사용자 설정 존중

## 사용 가이드

### 계층 구조 만들기

```tsx
// 명확한 정보 계층
<h1 style={textStyles.title.lg.value}>페이지 제목</h1>
<h2 style={textStyles.heading['2'].value}>섹션 제목</h2>
<p style={textStyles.body.md.value}>본문 내용...</p>
<span style={textStyles.caption.value}>추가 정보</span>
```

### 일관된 스타일 적용

```tsx
// Panda CSS와 함께 사용
<p className={css({ textStyle: 'body.md' })}>
  본문 텍스트
</p>

// 직접 스타일 객체 사용
<h1 style={textStyles.display.sm.value}>
  디스플레이 제목
</h1>
```

### 반응형 타이포그래피

```tsx
// 화면 크기에 따른 크기 조정
<h1
  className={css({
    textStyle: ['title.sm', 'title.md', 'title.lg'],
    // 모바일: sm, 태블릿: md, 데스크탑: lg
  })}
>
  반응형 제목
</h1>
```

## 접근성 고려사항

### 최소 크기

- **본문**: 14px (0.875rem) 이상
- **캡션**: 12px (0.75rem) 이상
- **중요 정보**: 16px (1rem) 이상 권장

### 행간과 자간

- **긴 텍스트**: `lineHeight: 'balanced'` (1.5)
- **제목**: `lineHeight: 'tight'` (1.2)
- **자간**: 서체 특성에 맞게 조정

### 대비와 가독성

- **본문**: AA 기준 4.5:1 이상
- **제목**: AAA 기준 7:1 권장
- **색상**: Colors 가이드의 `fg` 토큰 사용

## 모범 사례

### ✅ Do

- 정의된 텍스트 스타일 사용
- 용도에 맞는 스타일 선택
- 일관된 계층 구조 유지
- 충분한 여백 확보

### ❌ Don't

- 임의의 크기나 굵기 사용
- 너무 많은 스타일 혼용
- 작은 크기에 얇은 굵기 조합
- 긴 텍스트에 좁은 행간 적용
